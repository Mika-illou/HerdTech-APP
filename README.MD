#  Farm Management Web App

## Description

Ce projet consiste à développer une application web de gestion de ferme** permettant de simuler les principales fonctionnalités d'un système de suivi et d'administration d'une exploitation agricole.

L'objectif est de reproduire une interface moderne et intuitive permettant la gestion du cheptel, du stockage, du suivi des activités, des tâches et des notifications liées à la ferme.

Ce projet a également pour but de familiariser les étudiants avec l'utilisation des fichiers **SVG**, l'intégration de ressources externes et l'adaptation de code provenant de dépôts GitHub.


## Objectifs Pédagogiques

* Concevoir une application web simulant un programme de gestion de ferme.
* Découvrir et manipuler les fichiers **SVG**.
* Utiliser la balise HTML `<svg>` pour l'intégration d'icônes vectorielles.
* Intégrer et adapter des ressources externes provenant d'un dépôt GitHub.
* Développer une interface responsive en utilisant principalement **Bootstrap**.
* Respecter une charte graphique et une maquette imposées.


## Fonctionnalités

### Tableau de bord

* Vue d'ensemble de la ferme.
* Accès rapide aux différentes sections.

### Carte

* Visualisation des emplacements et zones de la ferme.

### Gestion du cheptel

* Consultation des animaux.
* Suivi des mouvements et ajouts.

###  Gestion du stockage

* Surveillance des stocks.
* Gestion des ressources disponibles.

### Suivi

* Contrôle des activités et opérations de la ferme.

### Gestion des tâches

* Affichage des tâches planifiées.
* Organisation des activités quotidiennes.

###  Notifications

* Notifications de routine.
* Notifications de vérification.
* Notifications de danger.
* Notifications critiques.

## Responsive Design

L'application est entièrement responsive et s'adapte aux :

* Smartphones
* Tablettes
* Ordinateurs de bureau

Grâce au système de grille Bootstrap.


##  Consignes de Réalisation

* Respecter les couleurs et les proportions de la maquette fournie.
* Utiliser Bootstrap pour la majorité du design.
* Limiter le CSS personnalisé à moins de 0 % du code total.
* Utiliser la balise `<svg>` pour les icônes spécifiées dans les consignes.
* Respecter l'organisation et l'ergonomie de l'interface.


##  Ressources Utilisées

Le projet s'appuie sur :

* Un dossier Google Drive contenant les icônes SVG et les images.
* Un dépôt GitHub fournissant les composants Material Input à intégrer et adapter.
